# Page Frameworks Reference

CRO frameworks for specific page types. For copy/messaging fixes, delegate to Ghostwriter. For UI/layout fixes, delegate to Seurat.

---

## Homepage CRO

Homepages serve multiple audiences simultaneously.

### Key Challenges
- Cold visitors need clear positioning
- "Ready to buy" and "still researching" visitors coexist
- Multiple conversion paths compete for attention

### Framework
1. **Clear positioning statement** that works for first-time visitors
2. **Quick path to primary conversion** (visible without scrolling)
3. **Self-selection navigation** — help visitors find their path
4. **Trust signals** — logos, stats, testimonials visible early
5. **Secondary paths** — for visitors not ready to convert

### Experiment Ideas
- Headline: specific vs abstract, benefit vs feature
- Hero visual: screenshot vs GIF vs illustration vs video
- CTA button text, color, and placement
- Customer logo placement: hero vs below fold
- Sticky nav with CTA vs standard navigation
- Live chat widget vs AI chatbot

## Landing Page CRO

Single-purpose pages. Focus on one conversion.

### Framework
1. **Message match** — headline must match the ad/source that brought them
2. **Single CTA** — remove navigation, sidebar, competing links
3. **Complete argument** — everything needed to convert on one page
4. **Above-fold completeness** — headline, subheadline, CTA, trust signal all visible
5. **Urgency/scarcity** — only if genuine

### Above-the-Fold Checklist
| Element | Requirement |
|---|---|
| Headline | Clear value proposition |
| Subheadline | Specific benefit or outcome |
| Hero image/video | Relevant, shows outcome |
| CTA | Prominent, action-oriented |
| Trust signal | Logo strip, rating, or stat |

### Common Issues
- Feature-focused instead of benefit-focused headline
- Value proposition too vague ("The best solution")
- CTA text is generic ("Submit", "Learn More")
- Stock photos that add nothing
- Missing trust signals near CTA

### Experiment Ideas
- Headline variations: outcome-focused vs problem-focused
- CTA copy: "Start Free Trial" vs "Get Started" vs "See Demo"
- Trust element placement: near CTA vs separate section
- Social proof type: customer logos vs testimonials vs stats
- Page length: long-form vs concise
- Video: product demo vs customer testimonial

## Pricing Page CRO

High-intent visitors. They're already interested — help them decide.

### Framework
1. **Clear plan comparison** — easy to scan differences
2. **Recommended plan** — visual indicator on target plan
3. **Feature clarity** — what's included and excluded
4. **Objection handling** — FAQ, guarantee, "cancel anytime"
5. **Easy path to checkout** — minimize steps from decision to payment

### Experiment Ideas
- Annual billing discount prominence
- Number of visible tiers (2 vs 3 vs 4)
- "Most Popular" badge placement
- Price anchoring strategy
- Feature comparison table format
- ROI calculator or value demonstration
- Testimonials specific to pricing/value
- Toggle monthly/annual with savings highlight
- "Contact Sales" vs showing enterprise pricing

## Feature Page CRO

Visitors researching specific capabilities.

### Framework
1. **Feature → Benefit connection** — what it does AND why it matters
2. **Use cases** — concrete examples of the feature in action
3. **Comparison** — how this compares to alternatives
4. **CTA** — clear next step (try it, demo, learn more)
5. **Social proof** — customers who use this specific feature

### Experiment Ideas
- Feature demo: video vs GIF vs interactive
- Benefit framing: quantitative ("save 5 hours/week") vs qualitative
- Comparison table: vs competitors vs no comparison
- CTA placement: inline vs end of page

## Blog Post CRO

Content-to-conversion bridge.

### Framework
1. **Contextual CTAs** — match CTA to content topic
2. **Lead magnets** — related to the article subject
3. **Inline CTAs** — at natural stopping points (not just end)
4. **Exit intent** — backup capture for leaving readers
5. **Content upgrades** — enhanced version of what they're reading

### Experiment Ideas
- Floating/sticky CTA vs inline only
- Inline CTA position: mid-article vs end
- Lead magnet type: checklist vs guide vs template
- Gated vs free content strategy
- Reading time indicator
- Related resources at end

## Cross-Page Patterns

### Visual Hierarchy
Every page should answer these questions through visual design:
1. **What is this?** (H1, hero)
2. **Why should I care?** (Benefits, proof)
3. **What should I do?** (CTA)
4. **Can I trust this?** (Social proof, guarantees)

### CTA Hierarchy
- One clear primary CTA
- Repeated at decision points (after benefits, after proof, end of page)
- Secondary CTA for not-ready visitors (lower commitment)
- Commitment level matches funnel stage

### Trust Elements Priority
```
STRONGEST (use at least 3):
├─ Customer logos (recognizable brands)
├─ Review score (4.5+ stars with count)
├─ Security badges (payment, compliance)
├─ Money-back guarantee
└─ Phone number visible

SUPPORTING:
├─ Testimonials (photo, name, company)
├─ Case study metrics
├─ "As seen in" media
├─ Team photos
└─ Live chat
```
