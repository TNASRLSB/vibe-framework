# Emmet — Knowledge Base

Domain knowledge and reference material for the Emmet testing/QA skill. This file is for human readers — Claude does not load it during skill execution.

---

## Directory Structure

```
emmet/
├── SKILL.md                # Main skill definition (loaded by Claude)
├── KNOWLEDGE.md            # This file (reference only)
├── prompts/
│   ├── map.md              # Workflow e logica di scan per /emmet map
│   └── test.md             # Workflow unificato per /emmet test
├── templates/
│   └── functional-map.md   # Template output per la map
├── testing/
│   ├── static.md           # Regole test statici
│   ├── dynamic.md          # Regole test Playwright
│   ├── plan-template.md    # [LEGACY] Template piano test
│   └── report-template.md  # Template report bug
├── checklists/
│   ├── code-review.md      # Checklist per code review
│   ├── pre-deploy.md       # Checklist pre-deployment
│   ├── refactoring.md      # Checklist per refactoring sicuro
│   └── security.md         # Checklist sicurezza
└── stacks/                 # Generato da /adapt-framework
    └── [stack-name]/
        ├── patterns.md     # Pattern idiomatici per lo stack
        ├── commands.md     # CLI commands per lo stack
        └── gotchas.md      # Errori comuni per lo stack
```

### Directory descriptions

- **`prompts/`** — Detailed workflow instructions for `/emmet map` and `/emmet test`. These are referenced by SKILL.md and loaded when the corresponding command is invoked.
- **`templates/`** — Output templates. Currently contains the functional map template that structures the output of `/emmet map`.
- **`testing/`** — Test execution rules. `static.md` defines patterns for static analysis, `dynamic.md` defines Playwright-based browser testing rules, and `report-template.md` structures bug reports.
- **`checklists/`** — Standalone checklists loaded by `/emmet checklist [type]`. Four types: code-review, pre-deploy, refactoring, security.
- **`stacks/`** — Auto-generated by `/adapt-framework`. Contains stack-specific patterns, commands, and gotchas tailored to the detected project stack (e.g., Node.js, Python, Rust).
