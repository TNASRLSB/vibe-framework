> SYSTEM OVERVIEW

What Is It

Two components working together to make Claude a reliable software engineer instead of a guessing machine.

Operational Framework

CLAUDE.md + .claude/docs/ — Process rules: investigate before implementing, track decisions, verify before asserting. The "operating system" that prevents Claude's worst habits.

Specialized Skills

.claude/skills/ — Domain knowledge modules that Claude loads on demand. UI design, dev patterns, security analysis, SEO, video generation, tech debt auditing.

> WHY THIS EXISTS

The Problem & The Fix

Claude has systematic failure modes. The framework mitigates each one with a specific mechanism.

Without Framework

  • Claims functions exist without checking
  • Starts coding before understanding
  • Forgets decisions from earlier
  • Re-implements existing code
  • Generates generic "AI slop" UI
  • Declares refactoring done prematurely

With Framework

  • Registry — verified external memory
  • Specs — think before implementing
  • Decisions log — never contradict past choices
  • Anti-duplication — search before creating
  • ui-craft — distinctive, accessible design
  • Migration tracker — verified completion

> EXECUTION FLOW

How It Works

The standard workflow that runs every session.

Boot

Claude reads CLAUDE.md, registry, decisions, and glossary. Understands what exists.

Investigate

Before any change, searches registry and codebase for existing solutions. No duplication.

Spec

For non-trivial changes, writes a spec. Waits for approval before writing code.

Build

Implements using existing patterns. Skills activate based on context.

Verify

Pre-commit checklist. Updates registry with new components.

Learn

Records decisions, session notes, failure modes. Memory persists.
